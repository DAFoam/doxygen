6  Description:
7 
8  This boundary condition provides an oscillating condition in terms of
9  (possibly multiple) amplitude and frequency.
10 
11  x_p = x_ref + a * sin(2 * pi * f * t + phase)
12 
13  where
14 
15  x_p : patch values
16  x_ref : reference value
17  a : amplitude
18  f : frequency [1/s]
19  t : time [s]
20  phase : phase in radian
21 
22  The input values are:
23 
24  Property | Description | Required
25  refVal | reference BC value | yes
26  amplitudes | oscillation amplitudes | yes
27  frequencies | oscillation frequencies | yes
28  phases | phase angles at zero | yes
29  endTime | oscillation end time (default 1e8) | no
30 
31  Example of the boundary condition specification:
32  myPatch
33  {
34  type multiFreqVector;
35  refValue 1.0;
36  amplitudes (0.5 1.0);
37  frequencies (10.0 5.0);
38  phases (0.0 1.0);
39  }
40 
41  This will add the oscillation with two frequencies to the refValue
42  1.0 at myPatch
43
