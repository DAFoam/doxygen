7  Description:
8 
9  NOTE: It's the nutUSpaldingWallFunction implementation in OpenFOAM-v2006,
10  which supports two additional inputs: maxIter and tolerance. In DAFoam,
11  we need to make sure this boundary condition provides as accurate as
12  possible boundary values, so the default values are maxIter = 1000
13  and tolerance = 1e-14. If this causes very slow simulations,
14  decrease maxIter or increate tolerance.
15 
16  This boundary condition provides a wall constraint on the turbulent
17  viscosity, i.e. \c nut, based on velocity, i.e. \c U. Using Spalding's
18  law gives a continuous \c nut profile to the wall.
19 
20  \f[
21  y^+ = u^+ + \frac{1}{E} \left[exp(\kappa u^+) - 1 - \kappa u^+\,
22  - 0.5 (\kappa u^+)^2 - \frac{1}{6} (\kappa u^+)^3\right]
23  \f]
24 
25  where
26  \vartable
27  y^+ | Non-dimensional position
28  u^+ | Non-dimensional velocity
29  \kappa | von Kármán constant
30  \endvartable
31 
32 
33  Usage
34  Example of the boundary condition specification:
35  \verbatim
36  <patchName>
37  {
38  // Mandatory entries (unmodifiable)
39  type nutUSpaldingWallFunction;
40 
41  // Optional entries (unmodifiable)
42  maxIter 100;
43  tolerance 1e-8;
44 
45  // Optional (inherited) entries
46  ...
47  }
48  \endverbatim
49 
50  where the entries mean:
51  \table
52  Property | Description | Type | Req'd | Dflt
53  type | Type name: nutUBlendedWallFunction | word | yes | -
54  maxIter | Number of Newton-Raphson iterations | label | no | 1000
55  tolerance | Convergence tolerance | scalar | no | 1e-14
56  \endtable
57 
58  The inherited entries are elaborated in:
59  - \link nutWallFunctionFvPatchScalarField.H \endlink
60 
61  Note
62  - Suffers from non-exact restart since \c correctNut() (called through
63  \c turbulence->validate) returns a slightly different value every time
64  it is called. This is since the seed for the Newton-Raphson iteration
65  uses the current value of \c *this (\c =nut ).
66  - This can be avoided by overriding the tolerance. This also switches on
67  a pre-detection whether the current nut already satisfies the turbulence
68  conditions and if so does not change it at all. This means that the nut
69  only changes if it 'has really changed'. This probably should be used with
70  a tight tolerance, to make sure to kick every iteration, e.g.
71  maxIter 100;
72  tolerance 1e-7;
